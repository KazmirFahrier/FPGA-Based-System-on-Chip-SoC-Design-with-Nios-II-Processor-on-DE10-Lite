# Hardware Design

This directory contains the Quartus and Platform Designer hardware portion of the DE10-Lite Nios II SoC.

## Files

| File | Description |
| --- | --- |
| `Embed.qpf` | Quartus project file. Open this file from Quartus Prime Lite. |
| `Embed.qsf` | Quartus settings, target device, source-file list, and board pin assignments. |
| `Embed.qsys` | Platform Designer system definition for the Nios II SoC. |
| `Embed.v` | Generated Verilog wrapper for the Platform Designer system. |
| `DE10_LITE_Default.sdc` | Timing constraints for board clocks and derived PLL clocks. |

## Target Board and Device

- Board: Terasic DE10-Lite
- FPGA family: MAX 10
- Device: `10M50DAF484C6GES`
- Top-level Quartus entity: `DE10_LITE_Default`
- Platform Designer system instance: `Embed`

## Major Platform Designer Blocks

| Block | Role |
| --- | --- |
| `nios2_gen2_0` | 80 MHz Nios II Gen2 processor. |
| `onchip_ram` | 16 KB internal RAM used for reset/exception code. |
| `onchip_flash_0` | MAX 10 internal flash interface. |
| `sdram` | External SDRAM controller. |
| `jtag_uart` | Debug console connection to the host PC. |
| `timer_0` | 32-bit timer used by HAL alarms. |
| `led_pio` | 10-bit output PIO for board LEDs. |
| `slide_pio` | 10-bit input PIO for slide switches, with edge capture interrupt support. |
| `spi_0` | SPI master for the board G-sensor connection. |
| `modular_adc_0` | MAX 10 ADC block, configured with channel 1 enabled. |
| `sysid` | Hardware/software identity check peripheral. |
| `mm_clock_crossing_bridge_0` | Avalon-MM bridge between CPU and peripheral clock domains. |

## Clocking

- `clk_0`: 50 MHz board input clock.
- `clk_1`: 10 MHz ADC-related input clock.
- `altpll_0`: generates 80 MHz clocks for the CPU/SDRAM side and a 40 MHz peripheral clock.
- `altpll_1`: drives the ADC PLL clock path.
- The SDC file creates the board clocks and derives PLL clocks for timing analysis.

## Rebuilding Hardware

1. Open `Embed.qpf` in Quartus Prime Lite.
2. Open `Embed.qsys` in Platform Designer.
3. Regenerate HDL if IP or interconnect settings changed.
4. Return to Quartus and run Analysis & Synthesis, Fitter, Assembler, and Timing Analysis.
5. Program the DE10-Lite using the generated SRAM Object File.
6. Regenerate the Nios II BSP before rebuilding firmware if any hardware addresses, names, clocks, or peripherals changed.

## Notes

- Keep the Platform Designer component names stable. The firmware validates expected peripherals through `software/system_validation.h`.
- If the LED PIO name, timer name, JTAG UART name, or SysID name changes, update the BSP and firmware validation header together.
- The generated `Embed.v` should normally be regenerated by Platform Designer rather than edited by hand.
